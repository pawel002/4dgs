Problem

Why naive per-frame reconstruction fails

Computational cost

Retraining each frame from scratch: 1 min of 30 fps video ⇒ 5.4 × 10⁷ optimizer steps — the static background is re-learned 1,800 times.

Lost correspondence

Free optimization gives no guarantee the same splat tracks the same physical feature — trajectories and motion analysis become impossible.

Colour-hallucinated motion

The loss is happy to repaint splats instead of moving them — subjects smear into the background as flat "paintings".

Storage growth

One independent Gaussian field per frame grows linearly with time × splat count — untenable for minutes-long captures.

~1.5 min. These four failure modes drive every design decision in the method. Highlight colour-hallucinated motion — it motivates freezing the SH coefficients later.
Background

Three eras of 3D reconstruction

Geometric

19818-point algorithm — two-view geometry in closed form
1992Factorisation — shape + motion from a rank constraint
2006Photo Tourism — SfM at internet scale
2016COLMAP — the SfM gold standard, still seeds every radiance field

Implicit neural

2020NeRF — scene as an MLP; photorealistic but ~1 day to train
2022Instant-NGP — hash grids; the MLP shrinks into a decoder

Explicit radiance fields

20233D Gaussian Splatting — the network disappears entirely; NeRF quality at >100 FPS
2024+4D extensions — the open frontier ← this thesis
~1 min, fast. The arc: geometry → networks → explicit primitives that keep differentiability. COLMAP survives as the initialisation everyone uses. The temporal domain is unsettled.
Background

3D Gaussian Splatting in 60 seconds

  • Scene = millions of soft ellipsoidal particles: position μ, covariance Σ = R S Sᵀ Rᵀ, opacity, spherical-harmonic colour
  • Rendered by splatting — project to 2D, depth-sort, alpha-blend in a tile rasterizer → >100 FPS
  • Trained end-to-end through a differentiable rasterizer, with adaptive clone / split / prune density control
  • Entire scene serializes to a single PLY file — inspectable, portable

The 3DGS optimization loop (Kerbl et al., 2023) — seeded by SfM points

~1.5 min. Key contrast vs NeRF: no ray marching, no MLP — rasterization, the operation GPUs were built for. Explicit primitives will be what lets us track motion per-splat.

Related work

Dynamic 3DGS today: ever more machinery

Deformable 3DGS

Canonical splat set + deformation MLP that must memorise every trajectory in the scene.

4D-GS

HexPlane spatio-temporal bases + decoder MLP — imposes a low-rank assumption on motion.

SpaceTime Gaussians

Per-splat polynomial trajectories — cheaper inference, enlarged parameter set per splat.

Each adds auxiliary machinery — with its own training dynamics and failure modes.
Is an almost machinery-free extension sufficient for indoor, static-camera captures?

~1 min. All three solve the general case (moving cameras, arbitrary scenes). Our regime is narrower: rigid background, sparse motion, fixed cameras — maybe simpler is enough.

Method

Proposed pipeline

COLMAP · frame 0 camera poses + sparse cloud computed once — cameras never move
→
Key frame full 3DGS fit · 2,000 iters densification on + dark-splat pruning
→
Tracking · every frame t velocity warm start → 500 iters topology frozen · colours frozen
→
Output one PLY per frame real-time 4D playback

The static–dynamic split is emergent: background splats receive ≈ zero gradient and stay put; splats on moving subjects are pushed along the motion — no segmentation model needed.

~1.5 min. Asymmetric budgets: keyframe establishes topology, tracking only propagates small geometric deltas (~25% of keyframe cost). Stress "emergent" — the optimization itself separates static from dynamic.
Method

Three constraints do all the work

1Frozen topology

No clone / split / prune after frame 0 ⇒ |𝒢ₜ| = |𝒢₀| and a 1-to-1 splat correspondence across the whole video.

2Frozen colours

SH coefficients locked after frame 0 ⇒ motion must be explained geometrically. Matches physics: reflectance is stable, position is not.

3Velocity warm start

Constant-velocity prediction before each frame ⇒ initial error is bounded by acceleration, not velocity — sub-pixel at 30 fps.

μₜ ← μₜ₋₁ + (μₜ₋₁ − μₜ₋₂)   — a single in-place tensor addition; zero extra parameters
~1.5 min. Constraint 2 directly answers the colour-hallucination failure mode from slide 4. Constraint 3 is first-order Taylor: the optimizer only pays for the acceleration residual.

Evaluation

A synthetic room with perfect ground truth

  • Full room modelled in Blender — varied materials, high-frequency textures, complex occlusion
  • Mathematically exact intrinsics & extrinsics exported via bpy — isolates 3DGS errors from SfM noise
  • Ideal pinhole camera — zero lens distortion
  • Drone-like oscillating orbit for uniform view coverage
~1.5 min. The parametric path: circular xy-orbit + fast sinusoidal height oscillation (ω slider). Mention: real-world evaluation is also planned — synthetic first to isolate the algorithm.
